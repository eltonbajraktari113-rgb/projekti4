# Menaxhimi i Kodit Burimor dhe Bashkëpunimi në GitHub

## Përshkrimi i Projektit
Ky projekt është zhvilluar për të demonstruar përdorimin e GitHub për menaxhimin e kodit burimor të një projekti softuerik të realizuar në gjuhën programuese C++.

Repository shërben si hapësirë e përbashkët për ruajtjen e kodit, bashkëpunimin mes anëtarëve të ekipit dhe menaxhimin e versioneve.

## Teknologjitë e Përdorura
- Gjuha programuese: C++
- Platforma për menaxhimin e kodit: GitHub
- Kontrolli i versioneve: Git

## Struktura e Projektit
- `main.cpp` – përmban kodin kryesor të aplikacionit
- `README.md` – dokumentacioni i projektit

## Menaxhimi i Kodit Burimor
Kodi burimor ruhet në një repository të përbashkët në GitHub. Çdo ndryshim regjistrohet përmes commit-eve të strukturuara, duke mundësuar gjurmimin e historikut të projektit dhe rikthimin në versione të mëparshme nëse është e nevojshme.

## Bashkëpunimi në Ekip
Zhvillimi i projektit realizohet në mënyrë bashkëpunuese. Anëtarët e ekipit përdorin branches për zhvillim paralel, duke shmangur konfliktet dhe duke ruajtur stabilitetin e degës kryesore `main`.

## Branches dhe Zhvillimi Paralel
- `main` – dega kryesore dhe stabile e projektit
- `feature-*` – përdoren për zhvillimin e funksionaliteteve të reja

## Push, Pull dhe Sinkronizimi
Anëtarët e ekipit kryejnë rregullisht:
- **Push** për të dërguar ndryshimet në repository
- **Pull** për të marrë versionin më të fundit të kodit

Kjo siguron që të gjithë punojnë me të njëjtin version të projektit.

## Pull Requests dhe Code Review
Përpara integrimit të ndryshimeve në degën kryesore, përdoren Pull Requests. Këto shërbejnë për rishikimin e kodit, diskutimin e ndryshimeve dhe miratimin e tyre, duke siguruar cilësi dhe menaxhim korrekt të kodit.

## Menaxhimi i Versioneve
GitHub ruan historikun e plotë të projektit, ku çdo commit përfaqëson një version të kodit. Kjo mundëson transparencë, kontroll dhe siguri gjatë zhvillimit të projektit.

## Përfundim
Përdorimi i GitHub në këtë projekt demonstron menaxhim profesional të kodit burimor, bashkëpunim efektiv në ekip dhe kontroll të saktë të versioneve të projektit.
